# Test Plan: WebSocket Integration

## 1. Подготовка окружения
- [ ] Убедиться, что сервер запущен на `http://localhost:8080`.
- [ ] Убедиться, что клиент запущен на `http://localhost:4173`.
- [ ] Проверить доступность WebSocket endpoint (`ws://localhost:8080/ws`).

## 2. Автоматизированное тестирование (Playwright)
- [ ] Создать тестовый файл `tests/e2e/websocket.spec.ts` (если не существует).
- [ ] Реализовать тест-кейс:
    1. Открыть страницу клиента.
    2. Ожидать появления индикатора статуса.
    3. Проверить текст/цвет индикатора (должен быть "Online" / зеленый).
    4. Проверить отсутствие ошибок в консоли браузера.
- [ ] Запустить тесты: `npx playwright test tests/e2e/websocket.spec.ts`.

## 3. Ручная верификация (Fallback)
- [ ] Открыть браузер по адресу клиента.
- [ ] Визуально подтвердить статус "Online".
- [ ] Открыть DevTools -> Console и проверить ошибки.
- [ ] Открыть DevTools -> Network -> WS и проверить статус соединения (101 Switching Protocols).

## 4. Завершение
- [ ] Зафиксировать результаты в `execution.md`.
- [ ] Сделать скриншот успешного подключения (опционально).
