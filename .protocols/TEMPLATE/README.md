# Template Protocol

Это шаблон для создания новых протоколов. Скопируйте эту папку и переименуйте её в формате `YYYY-MM-DD-feature-name`.

## Структура протокола

```
.protocols/YYYY-MM-DD-feature-name/
├── brief.md       # Требования и критерии приемки
├── plan.md        # План выполнения (Todo List)
├── execution.md   # Лог выполнения
└── context.md     # Дополнительный контекст (опционально)
```

## Как использовать

1. Скопируйте эту папку
2. Переименуйте её в `YYYY-MM-DD-feature-name`
3. Заполните `brief.md` с требованиями
4. Создайте `plan.md` с планом выполнения
5. Начните работу в Code Mode

## Примеры именования

- `2025-12-28-google-auth` - Добавление авторизации через Google
- `2025-12-29-fix-login-bug` - Исправление ошибки входа
- `2025-12-30-refactor-api` - Рефакторинг API
