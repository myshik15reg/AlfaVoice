# Memory Bank - AlfaVoice

> Централизованная база знаний о проекте для AI-агентов

## Quick Start

При старте сессии AI читает этот файл для понимания контекста проекта.

## Структура Memory Bank

| Файл | Назначение | Когда обновлять |
|------|------------|-----------------|
| [`brief.md`](brief.md) | Краткое описание проекта, основные цели | При изменении целей проекта |
| [`product.md`](product.md) | "Зачем?", цели продукта, проблемы пользователей, UX | При изменении видения продукта |
| [`architecture.md`](architecture.md) | "Как?", архитектурные решения, паттерны, структура | После архитектурных изменений |
| [`tech.md`](tech.md) | Стек технологий, зависимости, инструменты | При добавлении новых технологий |
| [`context.md`](context.md) | Текущий фокус работы, статус, следующие шаги | Регулярно, после каждой сессии |

## Навигация по режимам

### Architect Mode
**Читать:**
1. `brief.md` - понять цели
2. `product.md` - понять ценность
3. `architecture.md` - текущую архитектуру
4. `context.md` - текущий статус

### Code Mode
**Читать:**
1. `architecture.md` - паттерны кода
2. `tech.md` - технологический стек
3. `context.md` - текущие задачи

### Reviewer Mode
**Читать:**
1. `architecture.md` - стандарты архитектуры
2. `tech.md` - используемые технологии
3. `.kilocode/patterns/` - стандарты кодирования

## Принципы работы с Memory Bank

1. **Memory First** - всегда начинай с чтения этого файла
2. **Context Isolation** - работай только с релевантными файлами
3. **Update After Milestones** - обновляй после значительных изменений
4. **Keep It Concise** - информация должна быть лаконичной

## Интеграция с AlfaFlow

Memory Bank интегрирован с методологией AlfaFlow:
- Правила работы: `.kilocode/rules/`
- Протоколы задач: `.protocols/`
- Паттерны кода: `.kilocode/patterns/`
- Skills: `.kilocode/skills/`

## Команды

- `initialize memory bank` - инициализировать Memory Bank для нового проекта
- `update memory bank` - обновить все файлы Memory Bank
- `add task` - добавить повторяющуюся задачу в документацию
