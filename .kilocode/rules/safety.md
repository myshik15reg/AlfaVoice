# AlfaFlow Safety Rules

## Правила безопасности (Safety Rules)

1. **Никогда не удаляй** файлы из `.kilocode/memory-bank/` без явного подтверждения
2. **Всегда создавай резервную копию** перед изменением `architecture.md`
3. **Проверяй наличие активного протокола** перед любыми изменениями кода
4. **Обновляй `plan.md`** после каждого завершенного шага
5. **Записывай проблемы** в `execution.md` для будущего анализа
6. **Не коммить мусор**: Запрещено коммитить временные файлы, логи, отладочные артефакты или легаси код. Все временные файлы должны удаляться перед коммитом.

---

## Исключения (Exceptions)

**Допускаются изменения кода без протокола для:**
- Исправления опечаток в комментариях
- Форматирования кода (без изменения логики)
- Добавления логов для отладки
- Небольших улучшений документации

Во всех остальных случаях — создай протокол через Architect Mode.
